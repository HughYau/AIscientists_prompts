阅读指引（重要）

通用输出规范

  • JSON-only：要求结构化输出的提示，请确保模型“只输出 JSON”，不要附加任何说明或思维过程。
  • 单代码块：要求产出代码的提示，请确保模型“只返回一个 Python 代码块”，无需额外文字。
  • 占位符：使用时，必须用你的上下文替换花括号内容（如 `{Idea}` / `{page_limit}` / `{abstract}`）。

速用模板（严格格式）

引用检索 (Step 1)

必须直接返回 { "Description": "...", "Query": "..." }；若无新增，返回 {}。

引用选择 (Step 2)

必须直接返回 { "Selected": [...], "Description": "..." }；若不选，返回 {"Selected": [], "Description": ""}。

LLM 文本审稿

必须直接返回“REVIEW JSON”。

VLM 图像审稿

必须直接返回“REVIEW JSON”。

论文写作/聚合脚本

必须只返回完整代码块（`latex` 或 `python`），不加额外说明。

小贴士：若模型输出混入了解释性文字，请在你的提示中再次强硬地提醒它：“只输出 JSON”或“只输出代码块”。
